# Workflow: Phase 4 - 验证与迭代 (Verify & Iterate)

此工作流用于**质量保证阶段**。PRD输出不是终点，验证通过才能交付。

## 步骤 1: 自检清单 (Self-Check)

> **Goal**: 确保交付物符合产品质量标准。

请逐项检查以下维度：

### 逻辑完整性

- [ ] 状态流转是否闭环？（包含正向、逆向、异常路径）
- [ ] 异常处理是否覆盖？（网络超时、并发冲突、权限不足等）
- [ ] 业务规则是否编号？（R01, R02... 便于追溯）

### 数据规范性

- [ ] 字段类型是否精确？（金额用Decimal，状态用Enum，禁止模糊类型）
- [ ] 必填/可选是否明确？
- [ ] 枚举值是否列全？（如：状态码的所有可能值）

### 权限与安全

- [ ] 是否包含"权限说明"章节？
- [ ] 敏感数据是否脱敏？（手机号、身份证）
- [ ] 是否存在越权风险？（水平/垂直越权）

### 前端Demo（如有）

- [ ] **数据模型对齐** — 每个表单字段是否在 RDD 字段表中有对应？字段标签是否与 RDD 一致？
- [ ] **字段类型对齐** — Date 字段是否用了 `el-date-picker`？Enum 是否用了 `el-select`？Boolean 是否用了 `el-switch`？JSON 是否用了动态列表而非硬编码列？
- [ ] **实体数量对齐** — RDD 定义的 1:1 和 1:N 子实体是否都有对应的弹窗/Tab/表格？
- [ ] 新增按钮 → 是否打开弹窗？保存后是否写入数据数组、表格即时刷新？
- [ ] 编辑按钮 → 是否有 `@click` 处理？是否回填当前行数据？保存后是否替换数组项？
- [ ] 删除/禁用/启用按钮 → 是否有 `@click` 处理？是否真实修改数据？
- [ ] 弹窗保存 → 必填项是否有校验（非空提示）？校验失败是否阻止关闭？
- [ ] 筛选/搜索 → 下拉选择后表格数据是否过滤？
- [ ] 禁止项检查 → 是否存在只弹 toast 不写数组的保存？是否存在无 `@click` 的空按钮？是否存在文本框代替日期选择器？
- [ ] **标签闭合检查** → 是否存在 `<el-table-column .../>` 自闭合写法？CDN 模式下必须写成 `<el-table-column ...></el-table-column>`
- [ ] Mock数据是否符合真实业务场景？
- [ ] 是否覆盖异常状态的UI展示？（空状态、无匹配结果、错误提示）

## 步骤 2: 问题回溯 (Backtracking)

> **Goal**: 如果发现问题，回到对应阶段修正。

根据问题类型，回到对应Workflow：

| 问题类型         | 回到阶段        | 示例                           |
| :--------------- | :-------------- | :----------------------------- |
| **需求理解错误** | Phase 1         | 发现做的不是用户真实痛点       |
| **数据模型缺陷** | Phase 2 - 步骤1 | ER关系设计错误，缺少关键字段   |
| **边界情况遗漏** | Phase 2 - 步骤2 | 并发场景未覆盖，状态机有死循环 |
| **IA设计问题**   | Phase 2 - 步骤4 | 菜单层级不合理、页面遗漏       |
| **外部接口遗漏** | Phase 2 - 步骤3 | 缺少外部模块的接口依赖清单     |
| **PRD格式问题**  | Phase 3         | 缺少章节、格式不规范           |

**修正后，必须重新执行本阶段（Phase 4）验证**。

## 步骤 3: 最终交付 (Delivery)

> **Goal**: 确认无误后，正式归档。

1. 将文档从 `drafts/YYYY-MM-DD-[主题]/` 移动到 `prds/[模块名]/`。
2. 如果有前端Demo，确保代码可直接运行。
3. 输出《验收标准清单(AC Checklist)》供测试使用：
   - 基于Phase 1的RDD中的"验收标准"章节
   - 补充Phase 2发现的边界情况
   - 标注优先级（P0必测、P1重要、P2可选）

## 步骤 4: 用户确认 (User Confirmation)

询问用户：

1. PRD是否符合预期？是否需要调整？
2. Demo（如有）是否需要优化？
3. 是否可以正式交付给开发团队？

**Done**: 任务完成，进入开发排期！
