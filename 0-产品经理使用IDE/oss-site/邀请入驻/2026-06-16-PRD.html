1. 需求定义

1.1 背景与现状

销售与目标客户初步沟通后，发送专属入驻链接或邀请邮件。客户点击链接自助填写企业档案，提交后系统自动触发天眼查尽调并生成 PDF 报告存入客户资料。整个尽调环节无需销售手工操作。

1.2 目标与成功口径

  • 目标：客户提交入驻后系统自动完成尽调，销售零手工参与
  • 成功口径：客户提交入驻 → 尽调报告生成 < 30 秒，报告自动归档率 100%

1.3 范围与边界

  • In Scope（本期 P0）：邀请链接生成、邮件邀请发送、客户自助入驻（含 OCR）、系统自动尽调（工商+风险+PDF）
  • Out of Scope：销售手工输入企业名称查询、入驻审批工作流（二期）

3. 状态机

5. 页面规格

5.1 邀请入驻弹窗

页面信息：

  • 路径：客户管理列表 > 点击"发送入驻链接"
  • 类型：弹窗（500px），双 Tab 切换
  • 访问角色：销售代表

Tab 1 — 通过链接邀请：

字段名 中文名 类型 必填 默认值 备注
inviteLink 专属邀请链接 Input(disabled) ✅ 自动生成 格式 {base_url}/invite/{uuid}
expireDays 有效天数 Select — 14 7/14/30天

交互行为：

  • 点击"生成链接" → 创建 invitation_link 记录（token=UUID, expire_time=now+14天）
  • 显示链接 + "复制邀请链接"按钮
  • 复制时自动拼接前缀：诚挚的邀请您入驻{飞点/墨链}跨境供应链系统，点击链接处理：{link}

Tab 2 — 通过邮件邀请：

字段名 中文名 类型 必填 默认值 备注
recipientEmail 受邀者邮箱 Input(邮箱校验) ✅ — —
sender 发件人 Input(disabled) ✅ 系统默认 不可编辑
emailSubject 邮件主题 Input(disabled) ✅ 根据域名 "飞点/墨链跨境供应链入驻邀请"
emailBody 邮件正文 Textarea(disabled) ✅ 模板 占位符 {邀请链接} 发送前替换

交互行为：

  • 点击"发送" → 校验邮箱格式 → 替换模板占位符 → 发送邮件 → 创建 invitation_record 记录
  • 点击"返回" → 关闭弹窗

关联接口：

  • 生成链接：POST /api/invitation-link/create
  • 发送邮件：POST /api/invitation/email/send

5.2 企业客户档案页面

详细规格：见 客商中心 PRD §5.2（企业客户档案）。本章仅补充与邀请入驻相关的要点。

入口校验：

  • 校验 token 有效性和过期时间（14天）
  • 过期显示"链接已过期，请联系您的销售代表重新发送入驻邀请"
  • 已有暂存草稿 → 自动恢复表单

营业执照 OCR + 天眼查：

  • 上传营业执照 → OCR 自动识别公司名称/注册号/信用代码
  • 提交入驻后 → 系统自动调用天眼查 → 生成尽调报告 PDF
  • 天眼查失败 → Toast "企业风险信息获取失败，不影响入驻提交，稍后系统将自动补采"

关联接口：

  • 校验链接：GET /api/invitation-link/validate/{token}
  • 加载草稿：GET /api/profile/draft/{token}
  • 入驻提交：POST /api/profile/enterprise/submit（提交后自动触发尽调）

5.3 企业信息报告页面

页面信息：

  • 路径：客户管理 > 客户资料 > 企业信息报告
  • 类型：只读展示页，A4 PDF 排版
  • 访问角色：销售/客服/运营

报告内容（来自天眼查 API 自动填充）：

信息组 字段
基础信息 企业名称、统一社会信用代码、法定代表人、企业状态
注册信息 注册资本、成立日期、注册地址、经营范围
经营信息 参保人数、企业标签（高新技术企业/专精特新等）
评分 天眼评分（百分制，如 99.80）
数据溯源 数据来源"天眼查API"、查询时间

交互行为：

  • 页面为展示页，所有字段只读
  • 支持打印（@media print 隐藏操作按钮）
  • 页眉蓝色基调 #0066cc

5.4 企业风险报告页面

页面信息：

  • 路径：客户管理 > 客户资料 > 企业风险报告
  • 类型：只读展示页，A4 PDF 排版
  • 访问角色：销售/客服/运营

报告内容（来自天眼查 API 自动填充）：

区域 内容
头部 企业名称、风险总数
风险分类 自身风险 / 周边风险 / 预警提醒 / 历史风险
风险明细 风险等级标签（红/橙/蓝）、风险类型、标题、描述（含案号/金额/日期）

交互行为：

  • 风险等级颜色：高风险=红(#cf1322)、警示=橙(#faad14)、提示信息=蓝(#1890ff)
  • 某分类无数据 → 空状态"未发现该类别的风险信息"
  • 全部风险为0 → 空状态"未查询到任何企业风险记录"
  • 页眉红色基调 #d9363e

6. 业务规则

编号 触发点 条件/公式 输出 异常处理
R01 生成入驻链接 — 创建 invitation_link (token=UUID, expire=now+14天) —
R02 链接过期 now() > expire_time 客户点击提示"链接已过期" —
R03 链接已使用 status=已使用 不可再次使用 —
R04 客户提交入驻 enterprise_profile.status=20 触发天眼查 → 创建 due_diligence_log —
R05 天眼查查询 调用工商API + 风险API 填充 enterprise_info + enterprise_risk_report API失败→降级提示+补采
R06 尽调报告生成 查询成功 生成企业信息PDF + 风险PDF → 存入 customer_material —
R07 天眼评分转换 万分制 score 百分制 = score / 100，保留两位小数 —
R08 企业状态标签 存续/在业=蓝色，注销=灰色，吊销=红色 — —
R09 风险分类排序 自身→周边→预警→历史 固定顺序 —
R10 降级补采 lookup_status=30 定时任务每小时扫描 → 重试3次 → 仍失败告警 —
R11 邮件品牌切换 根据域名 飞点域名→"飞点跨境供应链"，墨链域名→"墨链跨境供应链" —
R12 复制链接前缀 — 拼接"诚挚的邀请您入驻{品牌名}跨境供应链系统，点击链接处理：{link}" —
