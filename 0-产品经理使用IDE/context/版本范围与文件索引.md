## 七、使用说明

1. 先看 §一 确认模块和对应文档
2. 按 **用户需求(RDD) → 数据设计 → PRD** 顺序阅读
3. 需求业务背景和辅助材料见 §五
4. 文件名日期 = 最近更新时间，帮助判断时效
5. 对照 §二 原型文件理解页面交互
6. PRD 为开发唯一真相源
