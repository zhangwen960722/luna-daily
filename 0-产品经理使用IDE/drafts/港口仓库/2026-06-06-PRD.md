### 1. 需求定义

**1.1 背景与现状**

跨境物流业务中，海外平台仓库(FBA/Walmart等)地址信息复杂，一个仓库代码对应多个物理地址，数据分散在Excel和即时通讯记录中。运单系统创建运单时需要引用仓库数据，但缺乏标准化的数据接口和唯一性校验，容易出现重复录入。国内本地仓的基础信息（收货人、手机号、地址）录入不规范，手机号无格式校验，收货人常漏填。海外仓库数据确认机制缺失，下游系统可能引用到未经核实的数据。

**1.2 目标与成功口径**

- 目标：为飞点TMS搭建仓库基础数据管理模块，实现本地仓和海外平台仓库的结构化管理，支持运单系统查询和新增
- 成功口径：海外仓库查询耗时从3-5分钟降至30秒内；仓库代码重复率从约10%降至接近0；手机号格式合规率100%

**1.3 范围与边界**

- In Scope（本期 P0）：港口管理(海港/空港，CRUD+禁用/启用)、海外平台仓库管理(FBA+其他平台，Tab切换、多地址、批量确认)、本地仓管理(Tab切换、CRUD+冻结/启用)、运单接口(查询+新增)
- Out of Scope：海外仓Tab实际内容（仓库列表页的海外仓Tab为空状态占位）、Excel批量导入、地址地图可视化、供应商专用地址标记（`isSupplierOnly`/`supplier`，经确认不纳入本期）、角色级权限管控（Excel源数据标注"无权限管控"）

**1.4 影响范围**

- 影响角色：仓库管理员、运单操作员
- 依赖系统：无外部系统依赖（本模块为被引用方，被运单模块依赖）
- 数据来源：Excel 表单 "平台仓库列表" + "仓库列表"

### 3. 状态机

**3.1 LocalWarehouse (本地仓) 状态流转**

```
[正常] ──{冻结}──> [已冻结]
  ^                  │
  └────{启用}────────┘
```

| 当前状态 | 操作 | 目标状态 | 触发角色 | 校验条件 |
|---------|------|---------|---------|---------|
| 正常 | 冻结 | 已冻结 | 仓库管理员 | 二次弹窗确认 (ElMessageBox.confirm, type:warning) |
| 已冻结 | 启用 | 正常 | 仓库管理员 | 二次弹窗确认 (ElMessageBox.confirm, type:warning) |

**3.2 OverseasWarehouse (海外平台仓库) 确认状态流转**

```
[未确认] ──{确认数据(批量勾选)}──> [已确认]
```

| 当前状态 | 操作 | 目标状态 | 触发角色 | 校验条件 |
|---------|------|---------|---------|---------|
| 未确认 | 确认数据 | 已确认 | 仓库管理员 | 1.至少勾选1条 2.勾选均为"未确认" 3.二次弹窗确认 |

> **说明**：确认状态仅单向流转（未确认→已确认），不支持从"已确认"回退到"未确认"。运单系统新建仓库时默认"未确认"。

**3.3 Port (港口) 状态流转**

```
[正常] ──{禁用}──> [已冻结]
  ^                 │
  └────{启用}───────┘
```

| 当前状态 | 操作 | 目标状态 | 触发角色 | 校验条件 |
|---------|------|---------|---------|---------|
| 正常 | 禁用 | 已冻结 | 仓库管理员 | 二次弹窗确认 (ElMessageBox.confirm, type:warning) |
| 已冻结 | 启用 | 正常 | 仓库管理员 | 二次弹窗确认 (ElMessageBox.confirm, type:warning) |

> **说明**：港口状态与本地仓状态一致，均为双向往返流转（正常↔已冻结）。已冻结港口编辑弹窗自动切换为只读模式。

### 5. 页面规格

| confirmStatus | 确认状态 | 标签 | overseas_warehouse.confirm_status | 已确认:success, 未确认:warning |

|--------|------|------|
| warehouseCode | el-input (200px) | 仓库代码，模糊搜索 |
| countryCode | el-select (200px) | 国家代码：US-美国/CN-中国/GB-英国/DE-德国 |
| confirmStatus | el-select (200px) | 确认状态：已确认/未确认 |

**按钮**：

| 按钮 | 说明 |
|------|------|
| 查询 | 执行搜索 |
| 重置 | 清空搜索条件 |
| 新增 | 按当前Tab打开对应新增弹窗 |
| 确认数据 | 批量确认勾选的"未确认"数据（需至少勾选1条+校验仅未确认行被勾选+二次弹窗） |

**字段表（新增/编辑弹窗 — 800px）**：

| 字段名 | 中文名 | 类型 | 必填 | 默认值 | 分组 | 备注 |
|--------|--------|------|------|--------|------|------|
| platform | 仓库所属平台 | input | ✅ | FBA Tab:"FBA"(disabled); 其他Tab:""(手动输入) | 基础信息 | FBA Tab下只读灰色 |
| warehouseCode | 仓库代码 | input | ✅ | — | 基础信息 | 提交时校验唯一性 |
| addresses[].countryCode | 国家代码 | select | ✅ | — | 地址信息 | US-美国/GB-英国/CN-中国/DE-德国 |
| addresses[].province | 州/省 | select | ✅ | — | 地址信息 | 根据国家联动(如US:CA/TX/NY; CN:GD/ZJ/SH) |
| addresses[].city | 城市 | input | ✅ | — | 地址信息 | — |
| addresses[].detail | 详细地址 | input | ✅ | — | 地址信息 | 街道门牌号 |
| addresses[].zipCode | 邮编 | input | ✅ | — | 地址信息 | — |
| addresses[].addressAbbr | 地址简称 | input(readonly) | — | 自动生成 | 地址信息 | 规则: warehouseCode + "-" + zipCode |

**交互行为**：
- [Tab切换]：FBA仓库列表 / 其他平台仓库列表。Tab切换时搜索条件和列表数据按Tab独立管理（或共用搜索区、仅列表过滤条件变化）
- [新增—FBA Tab]：点击"新增" → 弹窗标题"新增FBA平台仓库" → platform只读"FBA" → 默认含1个空地址行 → 支持"添加新地址"(追加行)
- [新增—其他平台Tab]：点击"新增" → 弹窗标题"新增其他平台仓库" → platform为空文本框 → 手动输入 → 其余同FBA
- [编辑]：点击行内"编辑" → 弹窗标题"编辑仓库" → 回显platform/warehouseCode/地址列表
- [添加地址]：弹窗内点击"添加新地址" → 新增一行空地址 → 至少保留1个地址(<=1时隐藏"删除地址"按钮)
- [删除地址]：点击地址行右上角"删除地址" → 从列表移除
- [地址简称联动]：watch warehouseCode + addresses[].zipCode → 实时更新 addressAbbr = warehouseCode + "-" + zipCode
- [仓库代码唯一性校验]：提交时前后端均校验 warehouseCode 是否重复 → 重复则 Toast "仓库代码已存在，请核对后重新输入" → 提交阻断
- [确认数据(批量)]：勾选行 → 点击"确认数据" → 校验：(1)至少1条 (2)选中行均为"未确认" → `ElMessageBox.confirm("确定要确认选中的N条仓库数据吗？","确认提示",{type:'warning'})` → 确认后选中行 confirmStatus 变为"已确认" → Toast"确认成功"
- [查询]：点击"查询"执行搜索
- [重置]：清空所有搜索条件

**关联接口**：
- 查询：`GET /api/overseas-warehouses` (params: warehouseCode, countryCode, confirmStatus, platform, page, pageSize)
- 新增：`POST /api/overseas-warehouses` (body: platform, warehouseCode, countryCode, dataSource, addresses[])
- 编辑：`PUT /api/overseas-warehouses/{id}` (body: platform, warehouseCode, addresses[])
- 批量确认：`PUT /api/overseas-warehouses/confirm` (body: ids[])

> **运单接口**（供运单系统调用）：
> - 查询：`GET /api/overseas-warehouses` (同上)
> - 新增：`POST /api/overseas-warehouses` (同上，新增时 confirmStatus 默认=10 未确认)

**交互行为**：
- [新增]：点击"新增港口" → 弹窗500px宽 → type默认"海港" → 输入字段校验 → 确定后Toast"新增成功" → 关闭弹窗，列表刷新
- [编辑]：点击行内"编辑" → 弹窗回显数据 → 若status=已冻结则弹窗标题变为"查看港口（已禁用）"，表单只读(is-readonly)，底部"返回"变为"关闭"，隐藏"确定"按钮
- [类型切换]：弹窗内type从海港切到空港（或反之） → portCode自动清空 → placeholder和maxlength同步变化
- [代码校验]：提交时校验：海港代码长度必须=5，空港代码长度必须=3；不符合时Toast提示"海港代码必须为5位"/"空港代码必须为3位"
- [禁用]：行内操作列显示"禁用"按钮（仅status=正常时显示） → 点击 → `ElMessageBox.confirm("确定禁用港口【{portName}】？","确认禁用",{type:'warning'})` → 确认后status变为"已冻结" → Toast"已禁用" → 操作列按钮切换为"启用"
- [启用]：行内操作列显示"启用"按钮（仅status=已冻结时显示） → 点击 → `ElMessageBox.confirm("确定启用港口【{portName}】？","确认启用",{type:'warning'})` → 确认后status变为"正常" → Toast"已启用" → 操作列按钮切换为"禁用"
- [查询]：点击"查询"执行搜索
- [重置]：清空所有搜索条件

**关联接口**：
- 查询：`GET /api/ports` (params: portName, portCode, type, page, pageSize)
- 新增：`POST /api/ports` (body: portName, portCode, country, type)
- 编辑：`PUT /api/ports/{id}` (body: portName, portCode, country, type)
- 状态变更：`PUT /api/ports/{id}/status` (body: status: 10正常/20已冻结)

### 7. 计算公式

**7.1 地址简称生成**

```
变量定义:
  warehouseCode = overseas_warehouse.warehouse_code（来源：表单输入）
  zipCode = overseas_warehouse_address.zip_code（来源：表单输入）

公式:
  addressAbbr = IF(warehouseCode != '' AND zipCode != '') THEN warehouseCode + '-' + zipCode ELSE ''

触发时机: warehouseCode 或 zipCode 变化时实时计算（前端 watch）
精度: 字符串拼接，无精度要求
条件分支: 任一方为空时不生成简称
```
