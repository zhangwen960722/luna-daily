# 箱唛设计要求（v2.0）

> **文档编号**: PRD-20260610  
> **创建日期**: 2026-06-10 | **修订日期**: 2026-06-13  
> **适用范围**: 跨境物流TMS系统 · 运单箱唛打印  

---

## 1. 概述

箱唛（箱贴/Box Label）是贴在货物外箱上的识别标签。本 PRD 定义 TMS 系统中运单箱唛的视觉设计方案、内容层级和字段取值逻辑。

**关键变更**：箱唛标识不再按"FBA/非FBA"定性，而是根据仓库所属平台代码动态显示。平台仓显示对应平台标识，私人地址显示通用标识。

---

## 2. 两种分类

| 分类 | 说明 | 标识来源 | 示例标识 |
|:---:|:----|:--------|:-------:|
| 🏭 **平台仓** | 发往平台仓库（Amazon FBA / Walmart / Mercado 等）的货物 | 仓库绑定的所属平台代码 | `Amazon` / `WALMART` / `Mercado` |
| 🏠 **私人地址** | 发往个人/私人地址的货物 | 固定为"非亚马逊运单" | `Personal` |

---

## 3. 视觉层级规范

箱唛统一采用 **10cm x 10cm** 方形尺寸，黑白打印，**不依赖颜色**区分信息层级。

### 3.1 高突出项（视觉权重最高）

| 字段 | 表现手法 | 说明 |
|:----|:--------|:-----|
| 平台标识 | 左上角显示(12px, 700字重) | 如 FBA · AMAZON / WALMART / Mercado |
| 运单号 | 右上角显示(20px, 800字重) | 如 FD260615479832 |
| 箱数 | 居中显示(32px, 900字重) | 如 1/10 |
| 目的国 | 目的地区块(26px, 800字重) | 如 US、MX、JP |
| 收件人 | 目的地区块(32px, 800字重) | 如 MQJ1、FTW1、Stephany |

**表现手法**: 区域化布局，按区域分层组织

### 3.2 中突出项

| 字段 | 表现手法 | 说明 |
|:----|:--------|:-----|
| 渠道名称 | (20px, 600字重) | 如 美国海派快线 |
| 昵称 | (20px, 800字重) | 如 jysj、qy-abfd |
| 邮编 | 加粗显示(20px, 800字重) | 如 91761 |
| 原产地 | 标准字号(16px, 600字重) | 默认 MADE IN CHINA |
| 报关方式 | (16px, 800字重) | 如 商业报关 |
| 内部追踪条码 | 不等宽条码(46px高度) | 1px窄条 / 3px宽条 交替 |
| 运单箱号 | 系统字体(18px, 600字重) | 如 FD260612478684U001 |
| 客户单号 | 系统字体(18px, 600字重) | 如 FBA19G240SPVU |
| 扩展箱号 | 系统字体(18px, 600字重) | 如 FBA19G240SPVU000001 |

**表现手法**: 按区域划分，区域4下划线标注，区域5等宽字体

## 4. 字段取值逻辑

| 字段 | 取值来源 | 说明 |
|:----|:--------|:-----|
| **昵称** | 客户关联的昵称 | 取客户资料中的昵称字段，如 jysj |
| **渠道名称** | 运单关联的渠道名称 | 取运单主数据中的渠道名称字段 |
| **目的国** | 运单地址信息 | 取运单收货地址中的国家代码 |
| **收件人** | 运单关联的收件人 | 取运单收货地址中的收件人名称，如 MQJ1、FTW1、Stephany |
| **平台标识** | 仓库绑定的所属平台代码 | 根据仓库代码关联的平台配置表，映射为显示标识：FBA → `Amazon`、WALMART → `WALMART`、Mercado → `Mercado` 等；若仓库未绑定平台，默认显示仓库代码 |
| **场景行** | 仓库绑定的平台代码 + 地址类型 | 平台仓根据平台代码映射场景描述：FBA `Amazon FBA Shipment`、WALMART `Walmart Fulfillment`、Mercado `Mercado Libre Fulfillment`；私人地址 `Personal Address Shipment` |
| **邮编** | 运单地址信息 | 取运单收货地址中的邮政编码 |
| **箱数** | 运单装箱单 | 分子为当前箱号，分母为总箱数，如 1/10 |
| **原产地标识** | 系统默认 | 默认为"Made in China" |
| **报关方式** | 运单报关信息 | 取运单报关信息模块中的报关方式字段 |
| **箱唛标识逻辑** | 平台代码 + 地址类型 | **平台仓**：根据仓库绑定的平台代码显示对应标识；**私人地址**：固定显示"非亚马逊运单" |
| **内部追踪条码** | 系统自动生成 | 系统根据运单号/箱号自动生成追踪码，呈现为Code39风格条码 |
| **运单箱号** | 装箱单 | 取装箱单中的运单箱号字段 |
| **客户单号** | 运单关联的客户单号 | 取运单主数据中的客户单号字段，如 7339751WFA |
| **扩展箱号** | 运单关联的扩展箱号 | 取装箱单中的扩展箱号字段，如 FBA19G240SPVU000001 |

## 5. 箱唛重新生成逻辑

当运单的货箱数量发生变更时，箱唛需要重新生成并重新贴标。

### 5.1 触发条件

| 操作 | 说明 | 处理方式 |
|:---|:----|:--------|
| 新增货箱 | 运单增加一个或多个货箱 | 重新生成该运单关联的 **全部** FD 箱唛 |
| 移除货箱 | 运单移除一个或多个货箱，剩余箱号出现断层 | 重新生成该运单关联的 **全部** FD 箱唛 |

### 5.2 原因说明

箱唛上的箱数格式为 `BOX x/y`，其中：
- x — 当前箱号（第几箱）
- y — 总箱数

当新增或移除货箱时：

1. **总箱数** — y 发生变化
2. **当前箱号** — 被移除货箱之后的所有箱号向前递补（如移除第2箱后，原第3箱变为第2箱），x 发生变化

因此运单关联的所有 FD 箱唛需要全部重新生成，确保箱唛信息与实际货箱一致。

### 5.3 重新贴标流程

1. 触发加箱/减箱操作
2. 系统自动重新生成该运单所有关联的 FD 箱唛
3. 原箱唛作废，仓库人员按新箱唛重新贴标
4. 新箱唛的条形码（内部追踪码）保持不变，仍关联原运单号

## 6. 布局规范

### 6.1 画布尺寸

- **尺寸**: 10cm x 10cm（方形标签）
- **打印适配**: 打印时自动缩放至 10cm x 10cm
- **分辨率像素**: 屏幕预览 378px x 378px（对应96dpi）

### 6.2 内容布局（从上到下）

箱唛从上到下共 5 个区域：

**区域1：运单号 + 箱数**
左边显示运单号（20px），右边显示箱数（32px），同行左右分布。
案例：FD260615479839    1/10

**区域2：服务渠道 + 昵称**
渠道名称与昵称左右并排（20px）。
案例：美国海派快线    jysj

**区域3：国家 + 收件人 + 邮编**
目的国（26px）+ 收件人（32px）+ 邮编（20px），左对齐排列，粗边框容器。
案例：US    MQJ1    91761

**区域4：原产地 + 报关方式**
原产地标识 Made in China（16px）+ 报关方式（16px）。

**区域5：条形码 + 箱号信息**
- Code39 风格条码（46px高度）
- 运单箱号（18px）
- 扩展箱号（18px）
- 客户单号（18px）
### 6.3 布局原则

- 使用 justify-content: space-between 自动均分垂直空间
- 条形码区域不设 flex-grow，保持固定高度
- 所有文字不使用底色填充，避免热敏打印糊墨
- 高突出项通过 **描边** + **粗边框** + **大字号** 实现

---

## 7. 打印规范

- **打印机**: 支持热敏标签打印机（如 Zebra、TSC 等）
- **颜色模式**: 纯黑白（灰度0-100%）
- **标签纸尺寸**: 10cm x 10cm（可定制）
- **页面设置**: 无边距打印
- **含 print CSS**: 打印时自动去除阴影、圆角、背景色
