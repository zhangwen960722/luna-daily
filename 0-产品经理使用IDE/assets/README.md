# 资源文件库 (Assets)

本文件夹用于统一管理项目中使用的各类静态资源文件，包括图片、图标、原型图、流程图等。

---

## 📍 使用场景

### 什么时候用这个文件夹？

- ✅ PRD需要插入流程图、原型图时
- ✅ AI生成了UI效果图，需要保存
- ✅ 要归档设计稿和系统截图
- ✅ 需要统一管理项目中的所有图片资源
- ✅ 演示PPT需要引用流程图、架构图

### 为什么需要assets文件夹？

> [!NOTE]
> 集中管理资源文件，避免图片散落在各处。所有PRD、分析文档、演示材料都可以引用assets中的资源。

## 📁 文件夹说明

### images/

通用图片素材，包括：

- PRD文档中引用的截图
- 界面效果图
- 业务场景示意图
- 数据可视化图表
- 参考图片

**建议子分类**：

```
images/
├── screenshots/    # 系统截图
├── ui-samples/     # UI示例
├── charts/         # 图表
└── references/     # 参考图片
```

### icons/

图标资源，包括：

- 功能图标
- 状态图标
- 导航图标
- UI组件图标

**格式建议**：

- 矢量格式：SVG、AI
- 位图格式：PNG（透明背景）

### mockups/

原型图和设计稿，包括：

- 低保真原型图
- 高保真设计稿
- 交互原型
- 设计规范

**工具来源**：

- Figma导出
- Sketch导出
- Axure导出
- 手绘草图扫描

### diagrams/

流程图和架构图源文件，包括：

- 业务流程图
- 系统架构图
- ER图（实体关系图）
- UML图
- 思维导图

**格式说明**：

- 源文件：`.drawio`、`.xmind`、`.eddx`等
- 导出图片：PNG、SVG
- Mermaid代码：`.mmd`文件

### 统一格式

`[模块名]-[内容描述]-[版本号].[扩展名]`

- `订单管理-列表页截图-v1.png`
- `库存中心-数据流转图-v2.png`

- `采购流程-业务流程图-v1.drawio`

## 🎨 资源使用规范

### 在Markdown中引用

**绝对路径引用**（推荐）：

```markdown
![订单列表页](file:///Users/mpp/Documents/AI Writing/产品经理使用IDE（进阶与实战）/assets/images/订单管理-列表页截图-v1.png)
```

**相对路径引用**：

```markdown
![订单列表页](../assets/images/订单管理-列表页截图-v1.png)
```

### 图片规格建议

**截图类**：

- 格式：PNG
- 分辨率：根据实际需求，建议不超过1920px宽
- 大小：压缩后不超过500KB

**原型图**：

- 格式：PNG（导出）、Figma/Sketch（源文件）
- 分辨率：1x或2x
- 背景：根据需求选择透明或白色

**流程图**：

- 格式：SVG（矢量）或PNG（位图）
- 清晰度：确保文字可读
- 源文件：保留可编辑的源文件

## 🔄 资源管理流程

### 1. 资源收集

- AI生成的图片
- 设计师提供的设计稿
- 原型工具导出的文件
- 系统截图

### 2. 资源整理

- 统一命名
- 分类存放
- 压缩优化
- 版本标注

### 3. 资源引用

- 在PRD中引用assets中的资源
- 使用绝对路径避免路径问题
- 注明图片说明和版本

### 4. 资源更新

- 更新时保留旧版本或修改版本号
- 在AGENT.md中记录重要资源的变更

---

## 💡 使用建议

1. **集中管理**：所有PRD、分析文档中用到的图片都放在assets中
2. **避免重复**：相同内容的图片不要重复保存
3. **定期清理**：定期清理不再使用的资源
4. **版本控制**：重要资源保留版本历史
5. **格式优化**：图片适当压缩，平衡清晰度和文件大小
