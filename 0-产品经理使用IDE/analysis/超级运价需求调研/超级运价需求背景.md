> **关联文档**：需求分析输出 → [RDD v3.4](../../drafts/2026-05-25-超级运价需求分析/RDD.md)（2026-05-27）

### 1.1 业务发展现状
我司（飞点跨境物流）经过8年发展，已从小规模货代，发展成了亚马逊FIST承运商。当前采用市面上新智慧的SAAS物流系统。目前的新智慧TMS系统，仅仅能承载一些业务数据，并不能做数据处理，且现有功能不满足实际的业务场景，甚至可以说，他等同于一个线上的超级记事本。随着业务的快速扩张，当前系统不满足实际需求，现在要从0-1开始搭建TMS跨境物流系统。整个供应链系统分为：TMS、WMS、CRM、SRM。我们目前需要梳理的是TMS-超级运价板块。

#### 1.1.1 亚马逊 FIST 承运商绩效考核

我司作为亚马逊 FIST 承运商，亚马逊会对我司进行关键指标考核。绩效考核对象为推送至亚马逊且在考核月份被亚马逊签收的北美（美国、加拿大）、欧洲（英法德意西）及日本线运单。

**考核指标：**

**1. 关键事件节点完整率**
- 定义：关键事件（干线出发 / 干线到达 / 海外清关 / 送仓）节点完整率 = 4 个事件均完整的货件 / 所有货件 × 100%
- FIST Elite 承运商要求：≥ 90%
- FIST 承运商要求：≥ 80%

**2. 事件节点维护及时率**
- 定义：事件发生时间与亚马逊系统抓取到对应事件的时间间隔在规定时效内即为及时。
- 各节点时效要求：
  - 揽收、干线出发、干线到达：事件发生后 **36 小时（1.5天）** 内被亚马逊系统抓取
  - 国内报关、海外清关、提柜拆柜、拿约：事件发生后 **60 小时（2.5天）** 内被亚马逊系统抓取
  - 送仓：事件发生后 **84 小时（3.5天）** 内被亚马逊系统抓取
- 计算公式：关键事件节点维护及时率 =（干线出发及时率 + 干线到达及时率 + 海外清关及时率 + 送仓及时率）/ 4
- 绩效要求：平均值 **> 80%**

**3. 尾程快递单号准确率**
- 定义：尾程为快递派的货件中，准确传输正确快递单号（可在快递公司官网查询到轨迹）的货件占比
- FIST Elite 承运商要求：≥ 90%
- FIST 承运商要求：≥ 80%

**4. 预计送达时段（DW，Delivery Window）准确率**
- 定义：在最后一次更新的预计送达时段内送达至亚马逊仓库的货件对应的商品件数，占卖家勾选"允许 FIST 承运商代为更新预计送达时段"货件对应的商品件数的比例
- FIST Elite 承运商要求：≥ 80%
- FIST 承运商如需启用 DW 自动更新功能，同样要求：≥ 80%

**5. 卖家授权比例（DW 自动更新启用率）**
- 定义：卖家允许 FIST 承运商代为更新预计送达时段的货件对应的商品件数，占所有 FIST 货件对应的商品件数的比例
- 卖家未授权的两种情况：(1) 创建货件时未选择"FIST 承运人"；(2) 未勾选"允许 FIST 承运商代为更新预计送达时段"
- 建议值：≥ 90%

> **对 TMS 系统建设的启示**：5 项考核指标中，关键事件节点完整率和维护及时率直接依赖 TMS 系统的节点数据采集和推送能力。当前外采系统（新智慧）无法实现事件节点的自动采集和向亚马逊的准时推送，这是自建 TMS 的重要驱动力之一。DW 预计送达时段准确率和卖家授权比例则与运单管理和客户沟通流程相关。

### 1.2 名词解释

#### 服务渠道相关
- 服务渠道：我司给客户提供的服务统称。
- 服务渠道分类：全段服务、分段服务
- 全段服务：指我司提供服务将客户的货，从中国运输到海外FBA仓或私仓。我们给客户提供的全段服务有"比如美森360、美森432、EX456"等。全段服务需配置服务范围。
- 全段服务需配置服务范围：干线运输方式、尾程派送方式、收费方式、包税方式、服务类型
- 干线运输方式：指货物从中国运输到提拆派仓的运输方式。支持：海运、空运、洲际火车、洲际卡车。我司目前主要开展海运、空运。
- 尾程派送：指从海外提拆派仓，派送到客户收货地。主流是：卡派、快递派。我司主要做海运-卡派。
- 收费方式：指货物的所有运输费用计算公式。我司支持：按KG、按CBM。
- 包税方式：指货物从中国到海外的税，是客户自己承担，还是需要我司完成交税提供全包服务。支持：包税、不包税、自主税号、自税递延。自税递延其实也是不包税的一种。
- 服务类型：指货物的装柜方式，若非大件，基本是走散货拼柜。若是大件，基本走整柜直送。目前我们提供的服务类型有散货、整柜。
- 整柜直送：指货物从中国到达海外港，清关提柜后，直接派送到目的仓，无需经过提拆派仓再分拣派送。
- 服务组合：根据全段服务+（服务范围）影响因素，组成的服务组合。举例：美森360-海运-卡派-KG-包税-散货、美森360-海运-卡派-CBM-包税-散货
- 分段服务：指我们可以将整个跨境运输流程中的节点，分节点提供服务。例如：订舱服务，当同行没有某船司的订舱资质时，我司可提供代理订舱服务。**⚠️ 一期不做，业务模型待三期定型。**
- 国家：系统中涉及的国家字段（起运国、目的国、港口所属国）均通过调用基础资料-国家接口获取下拉列表，不在超级运价模块内单独维护国家主数据。

#### 运输物流相关
- 航线：指货物从出发港到目的港的物理运输路径，由船司经营，服务渠道引用。船司信息通过 `supplier_id` 引用 SRM 供应商管理模块（供应商大类"干线运输供应商"→明细"海运/空运"），不在超级运价内重复维护。一个服务渠道可关联多条航线，例如美森360 可配置"上海→洛杉矶"、"宁波→长滩"两条航线。
- 航线代码：船司分配的航线标识，如 `AAC/AAC2`、`CLX`、`SEA/SEA3`。默认为空，可编辑。同一船司同一港口对可能存在多条航线代码（如 COSCO 宁波→长滩同时有 AAC/AAC2 和 AAC/AAC4/PS5），因此不能简单用出发港+目的港拼接。
- 出发港/目的港：指货物装船的起始港口和卸船的目标港口，如上海港（SHA）、洛杉矶港（LAX）。
- 预计运输天数：从出发到到港的预估天数，配置在航线级别，船期可覆盖。
- 周船期批次：按周汇总所有航线船期的主列表，是运价更新工作台的核心入口。每周一创建新批次（支持从上周复制），状态按 PREVIEW → CURRENT → EXPIRED 自动流转。运价查询时只展示 CURRENT 周的船期。
- 船期：一条航线在某一周的具体班次，挂在周批次下。包含截单时间（分仓库）、出发日期（ETD）、到港日期（ETA）、航程。运价行同样挂在周批次下，与船期同频每周复制+调价。例如"5月28日出发，6月15日到港，航程18天"。
- 截单时间：截止收货的时间，按本地仓库分别配置（如广州仓、汕头仓、义乌仓各有一个截单时间），超过此时间该仓库不再接收对应船期的货物。

#### 定价相关
- 成本底价：我们会对每个服务组合设置成本底价
- 公布价：根据运营维护的成本底价+利润策略+优惠政策，生成对外的公布价。
- 货主端：客户登录操作的端口。
- 运营端：我司登录操作的管理端。

#### 计费规则相关
- 计泡/方系数：体积换算重量的系数。当收费方式=KG时，此处一般填6000、5000。当收费方式=CBM时，此处一般填363。用于计算材重。
- 材重：材重=长(cm)×宽(cm)×高(cm)×件数/计泡系数，按每箱汇总后进位。
- 实重：实重=取红外线过机的实际重量
- 计重方式：决定收费重如何取值——材重（体积重）、实重（称重）、材重实重取大
- 收费重：若计重方式为材重实重取大，则根据服务组合绑定的计价方式(KG/CBM)决定比较逻辑。若为KG，实重和材重哪个大取哪个。若为CBM，重量方数和总体积哪个大取哪个。
- 计重精度：计费重量的进位精度，如 0.1KG、0.5KG、1KG
- 尺寸精度：长宽高的进位精度，如 0.1CM、1CM
- 最低箱收费重：每箱最低收费重量，不足按此值计费
- 最低票收费重：每票（整单）最低收费重量，不足按此值计费
- 最小件数 / 最大件数：该服务组合的起运和承运件数限制
- 重量方式：重量方式=实重/计方系数。按每箱汇总。
- 体积：单箱体积=长（米）*宽（米）*高（米），总体积按每箱汇总。

#### 成本计算相关（从 Excel 核算表挖掘）
- 成本段（全局目录，可自定义）：预置揽收段、干线运输段、关务段、仓储段、派送段，运价管理员可按需新增/禁用。服务渠道从全局目录中选择关联的段。
- 成本项模板（全局目录，挂成本段）：定义各段下可用的成本项（名称、建议默认价、币种、计算公式类型）。预置海运费、报关费、拆送价等常用项。渠道选择成本项时自动带出默认价，可覆盖。
- 渠道成本项：服务渠道从全局目录中选择段+项后，系统自动创建（纯数据层，无独立页面）。各渠道可为同一成本项设不同价格（如美森海运费 5.2 vs 快船 3.0），用于服务组合创建时继承和批量更新时追溯。
- 组合成本项：服务组合创建时从渠道成本项逐一复制。每个组合可独立覆盖单价、禁用不适用的项。成本底价计算时仅统计启用项。
- 计算公式模板（两种）：
  - 人民币直接计算：每公斤成本 = 原始单价 × 数量 ÷ 公斤标准装载量
  - 美元换汇计算：每公斤成本 = 原始单价 × 数量 × 汇率 ÷ 公斤标准装载量
  - 每方成本同理，分母替换为方数标准装载量
- 组合级公式参数（每个服务组合一套）：
  - 公斤标准装载量：该服务一柜装载 KG 数（如美森360为12525KG）
  - 方数标准装载量：该服务一柜装载 CBM 数（如美森360为75CBM）
  - 汇率：USD→RMB 汇率
- 派送段多来源报价（仅派送段）：供应商对散货给出组合一口价（按板）和拆送价（按仓点）两种报价方式，成本底价取两者的最大值（"哪个贵用哪个"）；整柜仅有直送价一个来源。
- 利润策略：在成本底价基础上加成，支持固定金额加成和百分比加成两种方式

### 1.3 痛点分析

#### 痛点1: 服务渠道无法体现公司战略
+ **现状**: 目前外采的TMS系统，只支持创建全段服务，分段服务无法体现出分段服务；
+ **问题**: 服务渠道无法区分全段服务、分段服务
+ **影响**: 业务发展模式受限，无法正常开展分段服务的业务。我司市场竞争力、影响力受限。
+ **一期策略**: 分段服务业务模式未定型，一期仅做全段服务的运价能力闭环。分段服务三期独立设计子模块。

#### 痛点2：成本→报价链路全部依赖手工 Excel
+ **问题**：
  - 140个服务组合 × 5段成本 × 每段3-5个细项，海运费每周变动时逐一换算 KG/CBM 成本价，一个组合约10分钟
  - 成本项缺少统一模板（币种/公式/默认价），每个渠道重复配置
  - 公布价靠运价管理员手工填，船期变了、成本变了容易忘记同步
  - **更深层问题**：成本底价靠手工计算（原始单价×汇率÷标准），汇率换算错误、公式引用错误风险高，且不可追溯
+ **影响**：运价更新效率低（全量 1 天），错误率高，老板质疑报价时说不清成本来源
+ **方案方向**: 成本四层模型（段→项模板→渠道→组合）+ 系统自动算 KG/CBM + 批量更新；公布价基于成本+利润策略自动生成

#### 痛点3：运价矩阵维度多，销售查询效率低
+ **问题**：
  - 公布价是仓库组×重量段×交货地的三维矩阵，KG/CBM 双计价，Excel 维护容易翻错行列
  - 船期信息在另一个 Sheet，销售被问"下周有没有船"时需切 Excel 查找
  - 优惠折扣（VIP/特批/折上折）靠人算，不同销售给同一客户报价可能不一致
+ **影响**：销售响应慢（客户群问 ONT8 这周什么价，要翻 Excel + 问运价管理员），报价一致性低
+ **方案方向**: 运价行挂周批次与船期同频 + 运价查询反向匹配引擎（输入货物参数 → 多渠道比价+船期+盈亏标识）+ 运费/附加费优惠独立规则引擎

#### 痛点4：Amazon FIST DW/时效考核无法系统化支撑
+ **问题**：
  - 亚马逊 FIST 承运商需向亚马逊推送预计送达时段（DW），当前手工维护无法自动推送
  - 运输时效、送仓时效、理赔承诺时效没有统一配置和校验
  - 延误理赔判定靠人工核对，缺乏系统自动化
+ **影响**：DW 准确率影响 FIST Elite 承运商考核；理赔漏判、误判无法追溯
+ **方案方向**: 服务渠道上配置预估运输时效+送仓预估时效（两阶段 DW 滚动更新）+ 理赔承诺时效

#### 痛点5：无法自动计算销售提成
+ **问题**：系统缺少销售提成的规则引擎功能；手工计算销售提成，效率低，错误高。
+ **影响**：效率低下，管理混乱。存在人工随意修改提成的隐患。
+ **归属判断**：❌ 不属于超级运价模块范围。归入 CRM 模块，超级运价仅需向其暴露利润数据 API。

#### 痛点6：无法自动获取供应商账单，自动分摊成本
+ **问题**：未通过接口形式获取供应商账单；群聊获取账单容易遗漏。
+ **影响**：手工分摊并导入成本，效率低下，错误率高。
+ **归属判断**：❌ 不属于超级运价模块范围。归入 SRM 模块。超级运价模块保留原始单价字段，便于二期与供应商账单核对。

#### 痛点7：无客户线上询价功能
+ **问题**：客户线下询价，存在销售乱报价，无法管理；无法收集客户高频询价渠道、仓点。
+ **影响**：通过群聊给客户报价，无法管控优惠政策。
+ **分期策略**: 线上询价 → 二期；一期聚焦运营端内部的运价配置和查询。

#### 痛点8：附加费手工判定，多数据源无法联动

+ **现状**：运单创建后，运价管理员需手工判断是否加收各类附加费。
+ **问题**：
  - 商品属性（粉末/液体/电池）需切换到商品管理模块查看 SKU 标签
  - 地址类型（商业/私人/偏远）需调商业接口，且不同尾程方式规则不同（卡派不收商业/私人，快递派全收）
  - 超重超长需等红外过机后才能计算（长+宽×2+高×2>265）
  - 客户自送、报关方式、本地仓等属性分散在不同表和接口
  - **核心问题**：没有一个统一的规则引擎把这些数据源串起来自动判定
+ **影响**：漏收、错收频发；新员工培训成本高（规则记不住）；无法追溯某笔附加费是根据哪条规则算的
+ **方案方向**：附加费规则引擎支持三种类型（字段匹配/公式阈值/人工添加），分节点触发（运单创建/红外过机），支持三种动作（加收/标识提醒/阻止下单），运价管理员可自行维护规则

#### 痛点9: 决策缺乏数据支持
+ **现状**: 大量Excel表格，缺少实时数据分析
+ **影响**: 决策盲目，错失市场机会
+ **分期策略**: 利润分析仪表盘 → 三期

### 1.4 需求目标（更新于2026-05-27）

#### Phase 1 — MVP（核心定价引擎）

**基础数据**：
- 服务渠道仅支持全段服务（分段服务预留 channel_type 字段，三期开放）
- 服务渠道上配置预估运输时效（FBA运单同步/DW推送）、送仓预估时效（两阶段 DW 滚动更新）、理赔承诺时效（按派送方式+FBA仓库）
- 重量段配置：服务渠道按 KG/CBM 分别定义计费分段阶梯
- 航线管理：航线创建（出发港/目的港/船司引用SRM），服务渠道 N:N 关联航线
- 周船期批次：按周创建/从上周复制船期+运价行，PREVIEW→CURRENT→EXPIRED 自动流转
- 船期维护：在周批次下录入各航线船期（截单时间分仓库/ETD/ETA/航程/最快提取），与运价同频更新
- 提单关联航线：提单绑定航线+船期，运单推送 Amazon FIST 时自动获取港口代码

**成本与定价**：
- 成本段 & 项模板（全局目录）：可自定义段+项（名称/建议默认价/币种/公式类型），预置 5 段+常用项
- 渠道成本项：服务渠道从全局目录选择段+项，继承默认价后可覆盖
- 服务组合自动生成（笛卡尔积），从渠道成本项逐一复制（默认启用），可禁用不适用的项 + 覆盖单价
- 系统根据公式 + 组合级参数（公斤标准装载量/方数标准装载量/汇率）自动计算各启用项的 KG/CBM 成本价
- 派送段独立支持多来源报价（组合一口价、拆送价按仓点、整柜直送价），成本底价取 MAX
- 利润策略配置：固定金额加成 / 百分比加成，预留 scope_type 支持段级/项级扩展
- 公布价自动生成（成本底价 + 利润策略），写入运价行，运价管理员可覆盖后发布
- 运价表/矩阵定价：运价行挂在周批次下，按 (服务组合, 仓库组, 重量段, 交货地) 四维维护，KG/CBM 双计价
- 计费规则：计泡系数/计重方式/计重精度/尺寸精度/最低收费重/最小最大件数，支持客户级覆盖
- 成本批量更新：修改成本项模板建议默认价 → 提示是否同步 → 展示受影响组合清单 → 勾选批量刷新
- 成本项 Excel 批量导入
- 成本项支持未来生效（effective_from 字段，一期手动切换）

**规则引擎**：
- 附加费规则引擎：三种规则类型（字段匹配/公式阈值/人工添加）+ 三种动作类型（加收费用/标识提醒/阻止下单）+ 两节点触发（运单创建/红外过机）；触发维度覆盖货物属性+运单属性（含 SKU属性/本地仓/用户/用户等级/地址类型/邮编/尾程方式等），多数据源联动；运价管理员可自行维护规则，支持关联渠道生效范围
- 运费优惠规则：按客户等级/客户特批配置，优惠方式可选折扣率或固定减免，支持折上折叠加 + 特批价生命周期管理（永久有效/固定期限/单次有效）
- 附加费优惠规则：绑定目标附加费规则，优惠方式可选折扣率或固定减免，与运费优惠独立计算

**查询与工作台**：
- 货主端在线询价：输入货物参数 → 多渠道比价+公布价+附加费（不展示成本底价和盈亏标识），匹配不到标"暂未报价"
- 运营端运价查询（销售视图）：输入货物参数 → 自动匹配可承运渠道 + 成本底价 + 含优惠最终报价 + 盈亏标识（绿/黄/红灯），响应 < 3s
- 价格底线管控：运价查询和特批价录入时展示盈亏标识，红灯弹窗确认（一期知会型）
- 运价更新工作台：本周船期+运价行同屏展示，船期/运价双 Tab，支持从上周复制

**审计与数据**：
- 操作审计日志：成本底价/运费优惠/附加费规则/特批价变更完整留痕，不可物理删除
- 询价日志：每次运价查询记录查询条件和返回结果，一期仅记录不做分析
