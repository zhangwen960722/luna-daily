# 超级运价模块 — 验收标准清单 (AC Checklist)

> 基于 RDD v3.4 + Phase 2 Architecture 风险报告 | 2026-05-28

## P0（必测，上线前必须通过）

| 编号 | 验收项 | 验收方式 | 通过标准 |
|------|--------|---------|---------|
| AC0a | 服务渠道创建 | 填写必填项→提交 | 起运国/目的国/运输方式/尾程/包税/计费/服务类型/needClaim 全必填校验 |
| AC0b | 预估运输时效 | FBA同步=是→填3字段 | 三个字段必填，填写完整后保存成功 |
| AC0c | DW滚动更新 | 运单创建→推送DW；提柜后→更新DW | 两个阶段推送的DW值分别计算正确 |
| AC0g | 目的国修改校验 | 减少目的国时检查FBA仓库 | 存在该国仓库配置时阻止操作并提示 |
| AC1 | 成本自动计算 | 录入原始单价+币种 | 与Excel手工计算结果偏差<0.01元 |
| AC2 | 组合自动生成 | 保存渠道并生成组合 | 笛卡尔积正确，渠道成本项逐一复制，默认启用 |
| AC6 | 批量更新 | 修改模板建议默认价→勾选组合→刷新 | 选中组合成本项单价更新，公布价1分钟内刷新 |
| AC7 | 附加费规则CRUD | 创建三类型规则各一条 | 字段匹配/公式阈值/人工添加分别能保存并匹配生效 |
| AC8 | 字段匹配型匹配 | 运单创建→自动匹配 | 条件组AND/OR正确，命中后按动作类型执行 |
| AC10 | 六种计价单位 | 创建规则，逐个切换计价单位 | 金额计算公式正确，精度0.01元 |
| AC13 | 运费优惠CRUD | 创建客户等级优惠+特批优惠 | 保存成功，查询时优惠自动应用 |
| AC15 | 特批价生命周期 | 创建固定期限→到期验证 | 到期自动过期，查询不再展示 |
| AC17 | 价格底线管控 | 设置低价→查询 | 绿灯(≥成本×1.05)/黄灯(薄利)/红灯(亏本)正确展示 |
| AC22 | 运价查询 | 输入条件→查询 | 响应<3秒，多渠道比价+船期+盈亏标识 |
| AC0o | 未匹配处理 | 查无报价的仓点 | 标记"暂未报价"并展示渠道，不屏蔽 |
| AC25 | 审计日志 | 修改成本底价→查日志 | field_name/old_value/new_value完整记录 |
| AC27 | 审计不可删除 | 尝试删除日志 | 无删除按钮/接口 |

## P1（重要，上线后一周内验证）

| 编号 | 验收项 | 验收方式 | 通过标准 |
|------|--------|---------|---------|
| AC4 | 派送段MAX | 拆送价+组合一口价同时存在 | 成本底价取MAX且可追溯来源 |
| AC9 | 公式阈值型匹配 | 红外过机→代入公式 | 多公式OR关系，任一命中触发 |
| AC11 | 人工添加型 | 运单界面勾选+填数量 | PER_UNIT×数量×单价计算正确 |
| AC14 | 优惠叠加 | 客户同时命中VIP+特批 | 折扣率乘法叠加，固定减免加法叠加 |
| AC24 | 货物参数匹配 | 输入件数+重量+长宽高 | 计费重量/计费体积计算正确，重量段匹配正确 |
| AC28 | 成本项未来生效 | 设置effective_from→手动切换 | 手动操作后成本价更新正确 |
| P1-Risk | 并发编辑冲突 | A和B同时编辑同一条运价行 | 乐观锁拦截，后提交者提示"数据已被他人修改" |
| P1-Risk | 删除被引用模板 | 删除已被渠道引用的成本项模板 | 阻止删除，提示引用数量 |
