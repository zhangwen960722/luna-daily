## 📥 输入 — 接到任务时

### 输入文件范围定义

### 工作模式判断 ★

> 加载文件前先判断模式。三种信号自动判断，无需用户显式声明。

**信号来源**：① 用户消息中的模块名（查 `context/版本范围与文件索引.md` §一）> ② IDE 打开文件路径中的 `drafts/[模块名]/` > ③ 对话历史中的模块上下文。

| 条件 | → 模式 | 加载范围 |
|------|--------|---------|
| 模块名不在版本索引 + 用户明确要新建 | 🌐 **新模块** | 版本索引 + `analysis/` 文件列表（可读的读，不可读的列文件名）；选读：从版本索引列所有模块实体摘要 → 识别可能数据交互的模块 → 问用户确认后加载 RDD |
| 模块名在版本索引 + 仅提 1 个 | 🎯 **已有模块** | 按 §任务→输入文件速查表，限定当前模块文件 |
| 提到 ≥2 个模块名 | 🔗 **跨模块** | 涉及模块的 RDD；选读涉及模块的数据设计 |
| 无法提取模块名 | ❓ **询问用户** | "针对哪个模块？" |

### 任务 → 输入文件速查表

> 确定模块后，按任务类型决定读该模块的哪些文件。

| 任务类型 | 典型话术（语义匹配） | 调用 Skill | 必读 | 补充读取 |
|---------|---------------------|-----------|------|---------|
| 需求分析 | 提出新想法、讨论方案、梳理流程："加个功能""能不能""怎么做""分析""梳理""优化""讨论""看看怎么改" | `requirement-clarifier` | RDD、需求背景 | 版本索引、Demo |
| 数据建模 | 问表结构、字段类型、实体关系："建表""ER图""字段类型""实体关系""主键""外键""索引""怎么设计" | `data-modeler` | RDD | 版本索引 |
| 边界探测 | 担心遗漏、异常、并发、安全："漏洞""边界""异常""并发""万一""会不会""检查一下""漏了什么""有问题吗" | `edge-case-detector` | RDD、数据设计 | 已有流程图 |
| 写 PRD | 方案定了要出正式文档："写PRD""出文档""定稿""归档""给开发的""整理成文档" | `prd-writer` | RDD、数据设计、现有 PRD + Demo | — |
| 生成/修改原型 | 要可见页面或改页面："画页面""生成Demo""原型""做页面""改页面""界面" | `frontend-designer` | RDD、数据设计、PRD、两端 index.html + 同端 ≥2 参考页 | — |
| 字段/实体变更 | 增删改字段、实体、表："加字段""删字段""改类型""加实体""拆表""字段名""枚举值""加个列" | — | RDD、数据设计、PRD | Demo 对应页面 |
| 流程/规则变更 | 改状态流转、业务规则、校验逻辑："状态流转""流程改""规则变了""R01""校验规则""条件改成""改成...状态" | — | RDD、数据设计、PRD | Demo 对应页面 |
| 页面调整 | 改现有页面的样式、布局、按钮、交互、文案："改颜色""改样式""加按钮""弹窗""布局""间距""文案""换成" | — | PRD、Demo 目标页 + 同端 ≥2 参考页 | RDD、数据设计 |
| 新增页面 | 要一个全新页面或功能模块："加页面""新建页""新页面""加个XX页""做个XX功能" | — | RDD、数据设计、PRD、两端 index.html | 同端 ≥2 参考页 |
**多任务命中**：合并所有命中行的文件（取并集）。**补充读取判断**：涉及 UI 变更 / 必读信息不足 / 用户要求对照 Demo 时触发，其他可跳过。

### 文件定位步骤

1. **判断模式**（§工作模式判断的 4 模式表）
2. **确定目标**：提取模块名 + 具体页面。模块未知 → 问；模块已知页面未知 → 问"涉及哪个页面？"
3. **查版本索引**：`context/版本范围与文件索引.md` → §一 定位文档路径；涉及原型 → §二 定位 HTML。若版本索引不存在（全新项目）→ 直接询问用户确认文件位置
4. **取最新日期**：同名多日期取最新。**若按日期拼接不存在**→ 列出实际文件后取最新，不要假设是今天
5. **读文件**：按模式 + 任务速查表加载

## ✏️ 文件操作 — 改文件时

### 修改前判断 ★

| 条件 | 操作 |
|------|------|
| 目标文件**不存在** | 新建 |
| 目标文件**已存在** | **先读后改**，在原文件上修改，不复制不另存 |
| 文件更新后 | 重命名日期前缀：`2026-06-01-用户需求.md` → `2026-06-07-用户需求.md` |

**原型 HTML 特殊规则**：页面文件始终在原文件上改，不改文件名。新增 → 新建 HTML + 注册到 `index.html`。删除 → 删 HTML + 从 `index.html` 移除。

### 修改后自检 ★

> 本回合文件操作全部完成后、回复用户前执行。

1. 回顾确认要改的内容（文件 + 改动点）
2. 核对实际改了哪些文件
3. 对比：确认的 ⊇ 实际的？
4. 遗漏 → 立即补
5. **文件索引同步 + 变更记录**：修改了设计文件（需求背景/RDD/数据设计/PRD/AC清单）→ 文件名日期更新为今天 + 同步更新 `context/版本范围与文件索引.md`。**新建了文件**（含 analysis/ 下的沟通文档等）→ 同步更新版本索引的对应章节。检查同模块文档内交叉引用链接是否失效（旧文件名→新文件名）。原型 HTML 不改前缀（豁免）

   **变更记录追加规则**：`drafts/[模块名]/变更记录.md`
   - 先读变更记录，检查当天是否已有同主题条目
   - 同日期 + 同主题（如"运价查询""工作台"）→ 合并到已有条目，不新增行
   - 新增条目时一句话概括业务变更（做什么 + 为什么），不加实现细节（不写 v2.0/v2.1 版本号、不列具体字段名）
   - 每天每个模块 ≤ 3 条
6. **联动检查**：改了设计文件（需求背景/RDD/数据设计/PRD/原型/AC List）→ 执行 §文件联动规则。联动产生的修改不触发新一轮自检
7. 输出：

```
📅 日期前缀：RDD(已更新至MM-DD) / 数据设计(未改) / PRD(已更新至MM-DD) / 原型(豁免)
📝 变更记录：[模块名] 变更记录.md | [变更内容一句话]
🔍 自检：确认改 N 个，实际改 N 个，全部命中 / ⚠️ 遗漏 X 个已补
```

### 文件联动规则 ★

> **触发**：本回合修改了设计文件（需求背景/RDD/数据设计/PRD/原型/AC List/index.html）。仅改变更记录/LOG/TODO/版本索引/CLAUDE.md 等元文件时不触发。

**硬规则**：改了任意一份设计文件 → 自动检查同模块其余设计文件中是否引用了被改内容（字段名、状态枚举、实体名、业务规则编号、页面引用等）→ 有则直接同步，不询问。

| 我改了 | 优先检查 |
|:---|:---|
| 需求背景 | RDD §核心洞察、PRD §需求定义 |
| RDD | 数据设计 §字段清单/§实体清单、PRD §业务规则/§功能清单、原型字段、AC List |
| 数据设计 | RDD §实体字段表、PRD §页面规格/§状态机/§枚举字典、原型字段 |
| PRD | RDD §验收标准、数据设计 §字段清单/§状态机、原型交互、AC List |
| 原型 | RDD §功能清单、数据设计 §IA设计、PRD §页面规格 |
| AC List | RDD §验收标准、PRD §验收标准 |

同步后输出：
```
🔗 联动同步：RDD(已同步§字段表) / 数据设计(已同步§字段清单) / PRD(无需) / AC List(已同步)
```

```
📋 LOG.md 已更新
```
